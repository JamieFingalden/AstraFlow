## 二、毕业设计开题报告

### 1. 选题背景与意义
- **选题背景**：
  - 随着数字化转型的深入推进，中小企业对财务管理系统的智能化需求日益增长
  - 传统财务管理方式效率低下，人工录入发票信息容易出错且耗时
  - 云计算和AI技术的发展为财务管理系统的创新提供了技术基础
  - 多租户架构在SaaS应用中的成熟应用为系统设计提供了参考

- **选题意义**：
  - **理论意义**：探索AI技术在财务管理领域的应用，研究多租户架构的设计模式
  - **实践意义**：开发实用的财务管理工具，提高中小企业财务管理效率
  - **技术意义**：实践前后端分离架构，掌握现代Web应用开发技术栈
  - **社会意义**：助力中小企业数字化转型，提升财务管理智能化水平

### 2. 国内外研究现状
- **国外研究现状**：
  - 发达国家在财务管理软件方面起步较早，如QuickBooks、Sage等产品已相当成熟
  - 国外企业级应用普遍采用SaaS模式，多租户架构技术相对成熟
  - AI技术在财务领域的应用相对领先，如机器学习用于风险评估、智能记账等
  - 但在OCR技术应用于发票识别方面，仍有较大提升空间

- **国内研究现状**：
  - 国内财务管理软件发展迅速，如用友、金蝶等传统厂商占据主要市场份额
  - 互联网企业也纷纷推出财务管理系统，如阿里巴巴的财务管家等
  - OCR技术在国内发展迅速，尤其在银行、税务等领域的应用较为成熟
  - 但中小企业专用的智能化财务管理工具仍存在市场空白

- **发展趋势**：
  - 云原生架构在企业应用中的普及
  - AI技术在财务管理中的深度应用
  - 移动端财务管理的持续发展
  - 数据安全和隐私保护要求的提升

### 3. 课题概述及主要技术路线、方法和手段
- **项目概述**：
  - 本项目旨在开发一个基于多租户架构的智能财务管理系统（AstraFlow），通过集成OCR技术实现发票自动识别与录入，为中小企业提供高效、智能化的财务管理解决方案
  - 系统采用前后端分离架构，支持租户隔离的SaaS模式，具备用户管理、发票管理、财务分析、报销流程管理等核心功能
  - 项目涉及前端开发（Vue 3 + TypeScript）、后端开发（Go + Gin）、AI技术集成（Python + OCR）等多个技术领域

- **主要技术路线**：
  - **前端技术路线**：
    - 采用Vue 3 + TypeScript构建现代化响应式用户界面
    - 使用Vite构建工具提升开发效率和构建性能
    - 引入Pinia进行全局状态管理，确保数据流一致性
    - 使用Axios进行HTTP请求处理，实现与后端API的交互
    - 数据可视化采用ECharts/Recharts，提供直观的财务数据分析

  - **后端技术路线**：
    - 采用Go语言 + Gin Web框架构建高性能RESTful API
    - 使用GORM进行数据库操作，支持MySQL等主流数据库
    - 实现JWT身份认证机制，保障系统安全性
    - 采用Gin框架中间件实现权限控制、日志记录、错误处理等通用功能
    - 通过RabbitMQ实现异步任务处理，提升系统响应性能

  - **AI技术路线**：
    - 使用Python Flask构建独立的OCR服务模块
    - 集成Tesseract OCR引擎进行发票文字识别
    - 应用OpenCV进行图像预处理，提升OCR识别准确率
    - 通过RabbitMQ消息队列与主系统通信，实现异步OCR处理
    - 设计发票信息提取算法，自动识别发票关键字段

  - **架构设计技术路线**：
    - 采用多租户架构设计，通过tenant_id字段实现数据隔离
    - 实现前后端分离架构，提升开发效率和系统可维护性
    - 使用Docker容器化部署，便于环境一致性和系统扩展
    - 采用微服务架构思想，模块化设计提升系统可扩展性

- **主要技术方法和手段**：
  - **软件工程方法**：采用敏捷开发方法，通过迭代式开发逐步完善系统功能
  - **数据库设计方法**：使用实体关系模型进行数据库设计，通过GORM实现数据访问层
  - **AI模型训练方法**：基于Tesseract预训练模型，针对发票识别场景进行优化
  - **系统集成方法**：采用API优先的设计理念，通过RESTful API实现系统间通信
  - **测试方法**：编写单元测试、集成测试，确保系统功能正确性和性能指标

## 三、毕业设计项目说明表

### 1. 项目简介
AstraFlow是一个基于多租户架构的智能财务管理系统，旨在为中小企业提供高效、智能的财务管理解决方案。系统通过集成OCR技术实现发票自动识别与录入，采用前后端分离架构，支持租户隔离的SaaS模式。项目涵盖用户管理、发票管理、财务分析、报销流程管理等核心功能模块，涉及前端开发（Vue 3 + TypeScript）、后端开发（Go + Gin）、AI技术集成（Python + OCR）等多个技术领域。

### 2. 项目意义和目标
- **理论意义**：探索AI技术在财务管理领域的应用，研究多租户架构的设计模式，为相关领域提供理论参考
- **实践意义**：开发实用的财务管理工具，提高中小企业财务管理效率，减少人工录入工作量
- **技术意义**：实践前后端分离架构，掌握现代Web应用开发技术栈，深度集成AI技术
- **社会意义**：助力中小企业数字化转型，提升财务管理智能化水平

**项目目标**：
- 构建高性能、可扩展的财务管理系统
- 实现OCR发票识别准确率≥90%
- 支持多租户架构，保障数据安全隔离
- 提供直观的数据可视化分析功能
- 实现完整的报销流程管理

### 3. 项目技术架构
#### 3.1 前端架构
- **核心技术**：Vue 3 + TypeScript + Composition API，利用响应式数据绑定和组件化开发模式
- **构建工具**：Vite，提供快速的冷启动和热模块替换功能
- **状态管理**：Pinia，轻量级状态管理库，支持TypeScript类型推导
- **路由管理**：Vue Router，实现单页应用(SPA)的路由跳转和管理
- **UI组件**：Element Plus（如使用），提供丰富的预设组件库
- **数据可视化**：ECharts/Recharts，实现财务数据的图表化展示
- **开发规范**：ESLint + Prettier，确保代码风格统一和质量

#### 3.2 后端架构
- **Web框架**：Gin，高性能HTTP Web框架，支持中间件机制
- **ORM框架**：GORM，Go语言的ORM库，支持多种数据库
- **数据库**：MySQL 8.0+，关系型数据库，支持事务处理
- **认证机制**：JWT（JSON Web Token），实现无状态用户认证
- **消息队列**：RabbitMQ，处理异步任务如OCR识别请求
- **日志管理**：Zap，高性能Go日志库，支持结构化日志
- **API文档**：Swagger，自动生成API文档，便于前后端协作

#### 3.3 AI模块架构
- **框架**：Flask，轻量级Python Web框架
- **OCR引擎**：Tesseract，开源OCR引擎，支持多种语言识别
- **图像处理**：OpenCV、Pillow，用于图像预处理和优化
- **任务队列**：与RabbitMQ集成，实现异步处理OCR任务
- **模型优化**：针对发票格式进行识别优化，提升准确率

#### 3.4 部署与运维架构
- **容器化**：Docker，实现环境一致性，便于部署与扩展
- **编排工具**：Docker Compose，管理多容器应用
- **负载均衡**：Nginx（如需要），处理高并发请求
- **监控指标**：Prometheus + Grafana（如需要），系统性能监控

### 4. 项目设计
#### 4.1 功能模块设计
- **用户管理模块**：
  - 用户注册、登录、注销功能
  - JWT身份认证与权限验证
  - 多租户数据隔离机制（tenant_id字段）
  - 个人用户与企业用户区分管理
  - 用户角色权限控制（admin、normal、personal）

- **发票管理模块**：
  - 发票文件上传（支持PDF、JPG、PNG等格式）
  - OCR自动识别发票关键信息（金额、日期、发票号等）
  - 发票数据分类与标签管理
  - 发票历史记录查询与统计
  - 发票图片预览与下载功能

- **财务分析模块**：
  - 收支统计与趋势分析
  - 数据可视化图表展示（柱状图、折线图、饼图等）
  - 月度/季度/年度财务报表生成
  - 预算管理与实际支出对比
  - 财务预警功能（超出预算提醒）

- **报销管理模块**：
  - 报销申请表单填写（关联发票）
  - 多级审批流程管理
  - 报销状态跟踪（待审批、已通过、已驳回等）
  - 报销记录统计与导出
  - 与财务数据的关联分析

- **系统管理模块**：
  - 租户管理（企业信息配置）
  - 系统配置参数设置
  - 操作日志记录与审计
  - 数据备份与恢复功能
  - 系统健康监控

#### 4.2 数据库设计
- 采用多租户架构设计，通过tenant_id字段实现数据隔离
- 用户表：存储用户基本信息、租户信息、角色权限
- 发票表：存储发票原始数据、OCR识别结果、关联信息
- 财务记录表：存储收入支出记录、分类统计信息
- 报销表：存储报销申请、审批流程、状态信息

#### 4.3 接口设计
- 遵循RESTful API设计规范
- 统一的响应格式：{code, message, data}
- 实现JWT认证和权限控制中间件

#### 5.1 前端实现
- 使用Vue 3 Composition API构建响应式界面

#### 5.3 AI模块实现
- 基于Tesseract OCR引擎实现发票文字识别
